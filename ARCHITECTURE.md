# Invincat CLI 架构文档

## 概述

Invincat CLI 是一个基于 deepagents-cli 二次开发的 AI 编程助手工具，提供交互式的终端界面，支持文件操作、代码编辑、shell 访问和 AI 代理功能。

## 架构设计原则

1. **模块化**: 功能分离，单一职责
2. **可测试性**: 易于单元测试和集成测试
3. **可扩展性**: 支持插件和功能扩展
4. **性能**: 异步处理和懒加载
5. **用户体验**: 直观的界面和交互

## 核心架构

### 1. 应用层 (Application Layer)

```
invincat_cli/app/
├── core.py              # 核心应用类 (DeepAgentsApp)
├── handlers.py          # 事件处理器
├── commands.py          # 斜杠命令实现
├── utils.py             # 应用工具函数
└── __init__.py
```

**职责**:
- 应用生命周期管理
- 用户界面渲染和交互
- 命令解析和执行
- 会话状态管理

**关键类**:
- `DeepAgentsApp`: 主应用类，继承自 Textual App
- `TextualSessionState`: 会话状态管理
- `AppResult`: 应用执行结果

### 2. 配置层 (Configuration Layer)

```
invincat_cli/config/
├── settings.py          # 全局设置类
├── bootstrap.py         # 环境引导和初始化
├── models.py            # 模型创建和管理
├── utils.py             # 配置工具函数
└── __init__.py
```

**职责**:
- 环境变量和配置文件管理
- API 密钥和模型配置
- 项目检测和路径解析
- 运行时配置

**关键类**:
- `Settings`: 全局设置单例
- 模型提供商工厂函数
- 配置验证和默认值

### 3. UI 组件层 (UI Components Layer)

```
invincat_cli/widgets/
├── messages/            # 消息显示组件
│   ├── base.py         # 基础消息类
│   ├── user.py         # 用户消息
│   ├── assistant.py    # 助手消息
│   ├── tool_call.py    # 工具调用消息
│   ├── diff.py         # 差异显示
│   └── __init__.py
├── chat_input.py       # 聊天输入框
├── status.py           # 状态栏
├── loading.py          # 加载指示器
└── ...                 # 其他组件
```

**职责**:
- 用户界面组件实现
- 消息渲染和格式化
- 交互事件处理
- 主题和样式管理

**关键组件**:
- `MessageStore`: 消息存储和管理
- `ChatInput`: 智能输入框
- `ToolCallMessage`: 工具调用显示
- `DiffMessage`: 文件差异显示

### 4. 工具层 (Tools Layer)

```
invincat_cli/
├── file_ops.py          # 文件操作工具
├── tool_display.py      # 工具显示格式化
├── mcp_tools.py         # MCP 工具集成
├── skills/              # 技能系统
└── ...
```

**职责**:
- 文件系统操作
- 工具调用跟踪和显示
- 外部工具集成
- 技能加载和执行

**关键模块**:
- `FileOpTracker`: 文件操作跟踪
- `format_tool_display`: 工具输出格式化
- `SkillLoader`: 技能加载器

### 5. 代理层 (Agent Layer)

```
invincat_cli/
├── agent.py             # 代理核心
├── textual_adapter.py   # UI 适配器
├── subagents.py         # 子代理管理
├── server.py            # 服务器管理
└── ...
```

**职责**:
- AI 代理执行和流式处理
- 工具调用协调
- 子代理创建和管理
- 服务器进程管理

**关键类**:
- `Agent`: 代理执行器
- `TextualUIAdapter`: UI 适配器
- `ServerProcess`: 服务器进程管理

## 数据流

### 1. 用户交互流程

```
用户输入 → ChatInput → 消息队列 → 代理执行 → 工具调用 → 结果渲染 → UI 更新
```

### 2. 工具调用流程

```
代理请求 → ToolCallMessage → 用户批准 → 工具执行 → 结果收集 → 差异计算 → 显示更新
```

### 3. 文件操作流程

```
工具调用 → FileOpTracker → 文件读取 → 修改执行 → 差异计算 → 预览显示 → 用户确认 → 文件写入
```

## 关键设计模式

### 1. 观察者模式 (Observer Pattern)
- 消息存储监听消息变化
- UI 组件响应状态更新

### 2. 工厂模式 (Factory Pattern)
- 模型创建工厂
- 工具实例化工厂

### 3. 适配器模式 (Adapter Pattern)
- Textual UI 适配器
- 后端协议适配器

### 4. 单例模式 (Singleton Pattern)
- 全局设置单例
- 消息存储单例

### 5. 策略模式 (Strategy Pattern)
- 不同模型提供商策略
- 不同工具执行策略

## 错误处理

### 1. 错误分类
- **用户错误**: 输入验证失败，配置错误
- **系统错误**: 文件权限，网络连接
- **运行时错误**: 工具执行失败，模型错误
- **逻辑错误**: 程序 bug，状态不一致

### 2. 错误处理策略
- **优雅降级**: 功能不可用时提供替代方案
- **重试机制**: 网络请求自动重试
- **用户反馈**: 清晰的错误消息和建议
- **日志记录**: 详细的错误日志和上下文

### 3. 恢复策略
- **状态回滚**: 文件操作失败时恢复原状
- **会话恢复**: 应用崩溃时恢复会话
- **配置回退**: 配置错误时使用默认值
